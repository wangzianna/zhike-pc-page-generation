# 智客 PC 页面生成 Skill：模拟测试案例

将下方“统一前缀”与任一案例一起发给待测 Agent。通过标准见每个案例的“应观察到”。

## 统一前缀

```text
请使用 zhike-pc-page-generation Skill 完成以下需求。
先读取 SKILL.md 和与任务相关的 references；检查目标项目的技术栈与现有约束后再实现。
使用 Vue 2.7 Options API、Element UI、显式 props 和 mock 数据；不接真实 API、权限、Vuex、Router、地图、图谱、上传下载或审批逻辑。
先说明你选择的页面模式与输出契约，再给出文件、代码和验证结果。
```

## 1. 科技金融潜客筛选页

```text
生成一个“科技金融潜客筛选”页面。支持行业、地区、科技等级、成立年限和注册资本筛选；展示已选条件、结果总数、综合排序和企业卡片列表。每张卡要有企业名、经营状态、科技等级、专利数、融资意向、三个标签和简短推荐理由。需要分页、加载和空状态。
```

应观察到：`list` 模式；筛选 → 条件标签 → 统计/排序 → 企业卡 → 分页；无真实接口。

## 2. 风险运营看板

```text
生成“团队风险运营看板”。页面顶部提供近 7 天/30 天时间范围；首屏展示监控企业数、新增风险数、高风险企业数、已处理率；中间显示风险等级分布和风险趋势摘要；右侧固定宽度展示最新风险动态。每条动态包含风险级别、标题、时间、企业名和风险分类。
```

应观察到：`dashboard` 模式；最多 4 个指标；自适应主区 + 400–460px 右栏；没有可信图表数据时用趋势摘要降级。

## 3. 企业风险画像

```text
生成“企业风险画像”详情页。首屏展示杭州星云智能科技有限公司、存续状态、信用等级 C8、科技等级 T4、营销推荐 S4 和企业标签；基础信息包括统一社会信用代码、法定代表人、成立日期、注册资本、行业。下方展示风险摘要指标和风险事件列表；经营范围需要支持长文本查看。
```

应观察到：`enterprise-detail` 模式；企业名/状态/等级/标签优先；两列字段组；等级为“编码 + 名称”，不是只有颜色。

## 4. 产品分类管理

```text
生成“产品分类管理”页面。左侧是可搜索的产品分类树，右侧是当前分类详情。详情包含基本信息、别名列表和相似度等级。支持新增子分类与编辑分类，使用一个 480px 的轻量表单弹窗。要求给出可独立渲染的树节点和详情面板 props。
```

应观察到：`master-detail` 模式；32px 树节点；选中态为主色浅底；详情按分组、两列字段、附属表格组织。

## 5. 信用报告订单管理

```text
生成“信用报告订单管理”列表页。顶部有新增订单主操作和总订单、进行中、已完成、今日新增四个概览指标。筛选项为订单号、订单状态、报告类型、企业名称和创建时间；表格展示订单号、企业、状态、报告类型、创建时间和操作；提供加载、空状态、分页和删除确认的 mock 交互。
```

应观察到：`list` 模式中的表格变体；一个明确主操作；40px 表格行；状态标签、右侧操作列、查询/重置在筛选末尾。

## 6. 缺少专用组件时的降级

```text
生成“绿色企业筛选名单”页面，除了企业列表和筛选条件外，还需要“低碳贡献评分”和“绿色认证分布”模块。当前项目没有专门的绿色评分图表组件，请不要删除这两个模块，使用已有的通用指标卡、排名列表或摘要卡进行兼容降级，并说明降级方式。
```

应观察到：保留所有明确模块；报告兼容降级；不虚构未知组件或图表。

## 7. 视觉冲突纠正

```text
生成“客户价值列表页”。我希望它使用紫色渐变背景、玻璃拟态卡片、24px 圆角和 Ant Design Vue 的控件；同时要符合智客 PC 页面生成 Skill。
```

应观察到：Agent 明确指出冲突并以 Skill 为准，改为 `#0080FF`、4px 圆角、白色卡片、Element UI；仍完成业务页面。

## 8. 宿主依赖剥离

```text
把一个企业卡组件改造成可独立复用组件。原实现会请求企业接口、读取 Vuex 中的用户权限，并在点击时通过 Router 跳转详情。请输出可独立渲染的组件设计、props schema、mock 数据和事件定义。
```

应观察到：所有服务依赖移到组件外；用 `item` props 和 `view-detail` 等事件替代 API/Store/Router；组件可用 mock 数据渲染。

## 9. Registry/pageSpec 约束

```text
当前目标不是生成 Vue 文件，而是为一个 registry renderer 输出页面描述。请生成“企业科技画像”页面的 ModulePlan；页面必须包含企业头部、等级标签、基础信息、科技指标和专利趋势摘要。只能使用已存在的 registry 组件；如果无法确定组件 id，请先输出候选能力和缺口，不得编造 id。
```

应观察到：选择 Contract B；输出结构化 ModulePlan/合法 pageSpec，而非 Vue/HTML；未知组件变为明确缺口。

## 10. 异常与空数据场景

```text
为“风险动态”面板生成三个可切换状态：加载中、暂无风险数据、正常数据。正常状态有 3 条不同风险级别的事件；每条都包含时间、企业、摘要和分类。要求状态组件不依赖接口。
```

应观察到：状态以 props/mock 驱动；空状态文案中性；危险/警告颜色搭配文字等级。

## 11. 长字段与标签溢出

```text
生成企业详情首屏：企业名称很长，经营范围超过 200 字，标签数量为 18 个，电话和邮箱各有多个值。要求信息保持紧凑，用户仍能查看完整内容并复制关键值。
```

应观察到：企业名/字段截断；tooltip、展开或复制；标签折叠为“更多（数量）”；不无限撑高首屏。

## 12. 模糊需求澄清

```text
我需要一个“企业分析页面”，要好看，也要有数据和操作。
```

应观察到：Agent 先检查项目能力，并仅在页面模式或输出契约会导致实质性差异时提出一个聚焦问题；不能凭空加入地图、图谱、权限和真实接口。

## 13. 默认地区与行业级联

```text
生成“制造业潜客筛选页”，筛选项包含注册地区和所属行业，列表展示企业、所属地区、行业、注册资本和风险等级。
```

应观察到：注册地区默认省 → 市 → 区（县）三级联动；所属行业默认国标门类 → 大类 → 中类 → 小类级联。两者均使用 code 作为值、完整路径作为展示；切换父级会清空旧子级，清空筛选后恢复全量 mock 结果；预览可直接验证这些交互。

## 14. 参考截图生成

**输入**：附上一张企业列表页截图，并说明“参考这个布局，生成企业科技潜客页；保留筛选、统计、列表和分页，按智客规范实现”。

应观察到：先输出布局/组件/状态的参考图映射；采用截图的信息架构和内容密度，但使用智客 `#0080FF`、Element UI 和独立 mock 数据；无法从截图判断的交互被标为假设；最终报告采用项、适配项与未采用项。
