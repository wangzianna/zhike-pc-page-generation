# 页面模式

## 高级筛选名单页

Order: range/base filters → condition groups → selected-condition chips → result count and sort → enterprise cards → pagination.

- Chips are removable, wrap, and have bounded height.
- Keep result count, sorting, batch actions, empty/loading states, and pagination.
- Enterprise cards show name, status, core score, tags, then key facts.

## 标准管理列表页

Order: title and one primary action → 4–6 summary metrics → filters → table → pagination.

- Use 40px table headers/rows, semantic status tags, and a right-side operation column.
- Put search/reset at the end of filters and keep secondary actions subordinate.

## 数据看板

Order: title/time filter → metrics → analysis → dynamic feed or ranking.

- On wide screens use flexible main content plus a 400–460px right rail.
- Metric cards are label → number+unit → description/trend.
- Downgrade unsupported charts to metrics, ranks, or trend summaries.

## 企业详情/画像

Order: enterprise name/status → grades/tags → core information → profile metrics → grouped detail/list sections.

- Use two-column key/value groups; prioritize credit code, legal representative, established date, registered capital, and industry when relevant.
- Collapse tags to “更多（数量）”; make long text inspectable or copyable.

## 主从管理

Order: search/actions → left tree/list → right detail panel → small edit dialog.

- Use 32px tree rows. Selected state is primary-tint background, primary text, and a light border.
- Detail panel: heading/actions → section headings → two-column fields → supporting table or empty state.
- Use a 480px dialog for simple edits.
