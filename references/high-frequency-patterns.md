# 高频企业筛选与结果列表

当需求是“潜客筛选、企业检索、企业库、找企业、名单筛选”或包含多类企业条件与企业结果列表时，优先使用本模式。它来自高密度企业筛选的交互意图，不绑定原项目接口或字段枚举。

## EnterpriseSearchFacetPanel

按下列顺序组织为一块无阴影的白色筛选区，区域之间用细分割线或留白区分：

1. **区域范围条**：当前地区/数据范围摘要、地区切换与可选的自定义区域入口。地区默认遵守省 → 市 → 区（县）三级联动。
2. **快捷检索条**：范围选择、企业关键词输入、查询按钮，以及可选的 AI 辅助入口。AI 入口只触发 `ai-assist` 事件，不实现真实 AI 服务。
3. **核心条件行**：所属行业、国标行业级联、新兴行业、商标/标签等高频条件；行业默认遵守门类 → 大类 → 中类 → 小类级联。
4. **密集条件组**：以“工商信息、企业评价、业务经营、金融市场、资金需求、资质证书、知识产权、风险过滤”等组名作为左侧短标签；同组内以紧凑文字触发器排列条件，带下拉标记。点击触发器打开局部 popover、drawer 或受控选择器，不直接展示宿主项目的复杂业务逻辑。
5. **展开控制与已选条件**：默认先展示核心且高价值的条件组；展开/收起不丢失已选值。若已选条件较多，在筛选区下方展示可移除 chips 与“清空”。

组件最小 props：`regionScope`、`keyword`、`coreFilters`、`conditionGroups`、`selectedFilters`、`expanded`、`loading`。事件至少包括 `change`、`search`、`reset`、`toggle-expanded`、`ai-assist`。筛选数据必须由调用方注入，预览使用嵌套 mock options 与本地过滤。

## EnterpriseResultList

把筛选结果放在白色结果区，而不是为每条企业制造独立大卡片。顶部为“结果总数 + 排序 + 批量关注/导出等次级操作”；主操作仅在任务确有需要时出现。

每条企业按下面的稳定层级排列，并用 `#EBEEF5` 或相近的细分割线分隔：

1. 企业全称、经营状态、信用/科技/营销等文字等级标签；右侧是关注或选择控制。
2. 企业类型、供应商/专精特新等业务标签；标签过多时折叠。
3. 法定代表人、成立日期、注册资本、统一社会信用代码等核心事实。
4. 电话、邮箱、官网、注册地址等联系信息；长地址和长字段必须截断并可查看完整内容。

组件最小 props：`items`、`total`、`sortOptions`、`selectedIds`、`loading`、`emptyText`、`page`、`pageSize`。事件至少包括 `sort-change`、`toggle-follow`、`selection-change`、`page-change`、`export`。每个 item 使用显式结构化字段，不在组件内请求企业详情。

## 视觉与状态

- 筛选区和结果区保持白底、4px 以下的轻量圆角或直角分区；不使用渐变、玻璃效果或装饰卡片。
- 筛选控件优先 28/32px 高度，密集条件文字使用辅助字号但保持可点击热区；结果总数强调数字，颜色使用主色 `#0080FF` 或风险语义色。
- 必须支持：默认核心条件、展开条件、已选条件、加载、无匹配结果和正常结果；筛选、排序、分页、关注/选择的 mock 状态保持同步。
