# 图片参考规则

当用户附上截图、原型图、竞品界面或其他页面图片时，将其作为设计参考，而非可直接复制的成品。

## 先提取，再实现

先给出简短的参考图映射，覆盖：

- 页面类型、主次区域、栅格/栏位与信息层级；
- 可复用组件、筛选/列表/详情/操作的可见状态；
- 可观察到的密度、间距、文字层级和视觉重心；
- 无法由静态图可靠判断的交互、数据或状态，标为保守假设，不得伪造。

仅有图片而没有文字需求时，也视为具体输入：根据图片推断页面模式，并说明一条必要的保守假设后继续。

## 适配优先级

按以下顺序决策：

1. 用户明确的业务需求与模块；
2. 本 Skill 的独立输出限制、组件规则和智客 Figma token；
3. 图片提供的布局、层级、组件组合与内容密度；
4. 图片中的装饰性细节。

当图片与智客规范冲突时，保留业务信息架构，按智客规范改造控件、颜色、圆角、间距和字体；明确说明冲突与适配结果。不得因图片出现而引入真实 API、权限、Router、Store、地图、图谱、上传下载或审批流程。

## 实现与预览

- 使用图片来指导布局和组件选择，不直接复刻第三方品牌、图标、图片资源或难以确认的文案；使用合法的占位资源、通用图标或 mock 文案。
- 保留图片中明确的业务模块；缺少专用组件时使用兼容降级，不得静默删除。
- 预览额外验证：主次区域、关键组件、内容密度和状态层级与参考图的意图一致，同时仍满足智客 token 与正常/加载/空状态要求。
- 最终报告列出“参考图采用项、适配项、未采用项及原因”。
